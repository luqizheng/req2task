# 预览版 v1.0 交付规格

## 1. 概述与目标

### 1.1 发布目标
交付 req2task 系统的**第一个可演示预览版本**，验证核心业务流程闭环。

### 1.2 演示范围
面向**内部评审**，验证以下核心场景：
- 用户认证与权限
- 项目与需求管理
- 任务看板管理
- AI 辅助需求生成

### 1.3 不在本次范围
- 通知系统
- 高级权限管理
- 完整测试覆盖

---

## 2. 功能规格

### 2.1 用户认证
| 功能 | 规格 | 优先级 |
|------|------|--------|
| 用户注册 | 邮箱、密码、昵称 | P0 |
| 用户登录 | JWT Token 认证 | P0 |
| 退出登录 | 清除 Token | P0 |
| 个人信息 | 查看和编辑 | P1 |

### 2.2 项目管理
| 功能 | 规格 | 优先级 |
|------|------|--------|
| 项目列表 | 分页展示所有项目 | P0 |
| 创建项目 | 名称、描述、状态 | P0 |
| 项目详情 | 查看项目基本信息 | P0 |
| 成员管理 | 添加/移除项目成员 | P1 |
| 基线管理 | 创建和查看基线快照 | P2 |

### 2.3 需求管理
| 功能 | 规格 | 优先级 |
|------|------|--------|
| 需求列表 | 分页、筛选、搜索 | P0 |
| 创建需求 | 标题、描述、优先级、状态 | P0 |
| 需求详情 | 查看完整需求信息 | P0 |
| 编辑需求 | 修改需求内容 | P0 |
| 需求状态流转 | 待办→进行中→已完成 | P0 |
| 用户故事 | 关联用户故事描述 | P1 |
| 验收标准 | 添加/编辑验收条件 | P1 |
| 需求变更日志 | 记录并展示变更历史 | P1 |

### 2.4 任务管理
| 功能 | 规格 | 优先级 |
|------|------|--------|
| 任务看板 | 按状态分组展示任务 | P0 |
| 创建任务 | 标题、描述、优先级、负责人 | P0 |
| 任务详情 | 查看任务完整信息 | P0 |
| 编辑任务 | 修改任务内容 | P0 |
| 任务状态拖拽 | 看板内拖拽改变状态 | P0 |
| 任务指派 | 分配负责人 | P1 |
| 任务依赖 | 查看任务前置依赖 | P2 |

### 2.5 AI 辅助功能
| 功能 | 规格 | 优先级 |
|------|------|--------|
| AI 配置 | 配置 LLM API Key | P0 |
| 需求生成 | 输入原始需求，AI 生成结构化需求 | P0 |
| 需求对话 | 对需求进行 AI 问答 | P1 |
| 任务分解 | AI 辅助将需求拆解为任务 | P2 |

### 2.6 仪表盘
| 功能 | 规格 | 优先级 |
|------|------|--------|
| 项目概览 | 展示项目统计数据 | P1 |
| 需求统计 | 需求数量、状态分布 | P1 |
| 任务统计 | 任务数量、状态分布 | P1 |

## 3. 技术规格

### 3.1 前端技术
- Vue 3 + Composition API
- TypeScript
- Vite 构建
- Pinia 状态管理
- Vue Router 路由

### 3.2 后端技术
- NestJS
- TypeORM
- PostgreSQL
- JWT 认证

### 3.3 部署方式
- 前端：本地开发服务器 (Vite Dev Server)
- 后端：本地开发服务器 (NestJS)
- 数据库：Docker PostgreSQL
