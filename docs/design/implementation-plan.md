# 实施路线图

## 1. 项目阶段规划

### 1.1 总体时间线

```
Sprint 1-2    Sprint 3-4    Sprint 5-6    Sprint 7-8    Sprint 9-10
    │             │             │             │             │
    ▼             ▼             ▼             ▼             ▼
┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
│  基础   │  │  核心   │  │  AI 1   │  │  AI 2   │  │  高级   │
│  设施   │→│  业务   │→│  能力   │→│  增强   │→│  功能   │
└─────────┘  └─────────┘  └─────────┘  └─────────┘  └─────────┘
```

### 1.2 里程碑

| 里程碑 | 时间 | 交付内容 | 验收标准 |
|--------|------|----------|----------|
| M1 | Sprint 2 | 基础 CRUD | 可创建项目、模块、需求、任务 |
| M2 | Sprint 4 | 核心业务闭环 | 需求状态流转、任务分配 |
| M3 | Sprint 6 | AI 基础能力 | 需求生成、RAG 检索 |
| M4 | Sprint 8 | AI 增强能力 | 冲突检测、上下文关联 |
| M5 | Sprint 10 | 高级功能 | 报表、看板、基线管理 |

## 4. 风险管理

| 风险 | 影响 | 概率 | 应对 |
|------|------|------|------|
| LLM API 不稳定 | 高 | 中 | 本地 Ollama 降级 |
| ChromaDB 性能问题 | 中 | 低 | 索引优化、分片 |
| 冲突检测准确率低 | 中 | 中 | 持续优化 Prompt |
| 需求变更 | 中 | 高 | 敏捷响应 |
