# 管理员权限

## 1. 项目管理员机制

### 项目创建者

当用户创建项目时，该用户自动成为项目的 **所有者（Owner）**。

| 属性 | 说明 |
|------|------|
| Owner | 项目的创建者和所有者 |
| 权限 | 项目最高权限，可管理所有设置和成员 |

### 自动关联

- 创建者的 `userId` 记录在项目的 `ownerId` 字段
- 创建者可对项目进行完整管理

---

## 2. 项目成员角色

### 角色类型

| 角色 | 说明 | 权限 |
|------|------|------|
| owner | 项目所有者 | 全部权限 |
| manager | 项目管理员 | 成员管理、配置管理 |
| developer | 开发人员 | 需求任务操作 |
| viewer | 访客 | 仅查看 |

### 角色说明

#### Owner（所有者）

- 项目创建者自动获得
- 不可转让
- 拥有项目全部权限

#### Manager（管理员）

- 可添加/移除项目成员
- 可管理项目配置
- 可创建/编辑模块和需求

#### Developer（开发人员）

- 可查看项目内容
- 可创建/编辑任务
- 可更新任务状态

#### Viewer（访客）

- 仅可查看项目
- 无修改权限

## 3. 成为项目管理员

### 方式一：创建项目

创建项目时自动成为项目所有者，获得最高权限。

```
1. 进入项目管理页面
2. 点击「创建项目」
3. 填写项目信息并提交
4. 自动成为该项目管理员
```

### 方式二：被添加为成员

被项目管理员添加为成员时，可获得指定角色。

```
1. 项目管理员进入项目详情
2. 点击「添加成员」
3. 选择目标用户
4. 完成添加
```

## 4. 权限继承

### 用户角色层级

```
系统角色（管理员/普通用户）
    ↓
项目角色（owner/manager/developer/viewer）
    ↓
模块/需求/任务操作权限
```

### 权限计算

1. 首先检查用户是否为目标项目的 owner
2. 检查用户在项目中的角色
3. 根据角色确定可进行的操作

## 6. 常见场景

### 场景一：新员工入职

1. 管理员创建用户账号
2. 将用户添加到相应项目
3. 分配适当角色（如 developer）
4. 用户即可访问项目

### 场景二：项目交接

1. 原 Owner 创建新项目（自动获得 Owner）
2. 将原项目数据迁移/复制
3. 添加原项目成员
4. 调整成员角色

### 场景三：外部人员协作

1. 创建用户账号
2. 添加到项目，分配 viewer 角色
3. 根据需要升级为 developer
