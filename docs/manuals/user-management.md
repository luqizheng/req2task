# 用户管理

## 1. 创建用户

### 首次注册说明

**重要**：系统第一个注册的用户将自动获得 **管理员（admin）** 角色。

这是系统内置的安全机制，确保系统始终有一个管理员账户。请使用第一个账户完成系统初始化配置。

### 注册流程

1. 进入登录页面，点击 **注册** 链接
2. 填写注册信息
3. 点击 **注册** 完成账户创建

### 自动角色分配

| 注册顺序 | 角色 | 说明 |
|----------|------|------|
| 第一个 | 管理员（admin） | 拥有系统全部权限 |
| 第二个及之后 | 普通用户（user） | 需由管理员分配角色 |

### 操作步骤

1. 进入 **用户管理** 页面
2. 点击 **添加用户** 按钮
3. 填写用户信息
4. 点击 **确定** 完成创建

### 必填字段

| 字段 | 说明 | 规则 |
|------|------|------|
| 用户名 | 登录账号 | 字母、数字、下划线，3-20位 |
| 姓名 | 显示名称 | 用于界面显示 |
| 邮箱 | 登录邮箱 | 有效邮箱格式，唯一 |
| 密码 | 登录密码 | 至少6位 |
| 角色 | 用户角色 | 系统角色分配 |

### 可选字段

| 字段 | 说明 |
|------|------|
| 头像 | 用户头像 URL |

### 角色说明

| 角色 | 说明 | 权限范围 |
|------|------|----------|
| 管理员 | 系统管理员 | 全部功能 |
| 项目经理 | 项目负责人 | 项目管理 |
| 需求分析师 | 需求分析人员 | 需求管理 |
| 开发人员 | 开发工程师 | 任务开发 |
| 测试人员 | 测试工程师 | 测试相关 |

### 操作示例

```
用户名：zhangsan
姓名：张三
邮箱：zhangsan@example.com
密码：********
角色：项目经理
```

## 2. 用户列表

### 列表功能

- **搜索**：支持按用户名、姓名、邮箱搜索
- **筛选**：支持按角色、状态筛选
- **分页**：支持自定义每页显示数量
- **排序**：按创建时间倒序排列

### 列表字段

| 字段 | 说明 |
|------|------|
| ID | 用户唯一编号 |
| 用户名 | 登录账号 |
| 姓名 | 显示名称 |
| 邮箱 | 登录邮箱 |
| 角色 | 用户角色标签 |
| 创建时间 | 账号创建日期 |
| 操作 | 编辑 / 删除 |

---

## 3. 编辑用户

### 可修改字段

- 姓名
- 邮箱
- 角色

### 操作步骤

1. 在用户列表点击 **编辑**
2. 修改需要更新的字段
3. 点击 **确定** 保存

### 密码修改

- 编辑用户时，密码字段留空表示不修改
- 如需修改密码，在密码字段输入新密码

---

## 4. 删除用户

### 注意事项

- 删除操作不可逆
- 用户数据将被永久移除
- 需谨慎操作

### 操作步骤

1. 在用户列表点击 **删除**
2. 在确认弹窗中点击 **确定**
3. 用户账号将被永久删除

---

## 5. 用户状态管理

### 状态类型

| 状态 | 说明 |
|------|------|
| 启用 | 正常登录使用 |
| 禁用 | 无法登录，但数据保留 |

### 状态切换

在用户列表中可通过筛选查看启用/禁用状态的用户。

---

## 6. 添加用户到项目

### 操作步骤

1. 进入目标项目的 **详情页**
2. 在 **成员管理** 卡片点击 **添加成员**
3. 选择需要添加的用户
4. 完成添加

### 成员角色分配

添加用户到项目时，用户在项目中的角色默认为基础角色。如需调整，可在成员管理中查看。
