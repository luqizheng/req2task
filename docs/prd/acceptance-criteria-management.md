# 验收标准管理

## 9.1 验收标准分类

### 功能验收标准
| 类型 | 描述 | 示例 |
|------|------|------|
| **正向场景** | 正常业务流程 | Given 用户已登录，When 点击保存，Then 数据保存成功 |
| **异常场景** | 错误处理流程 | Given 网络断开，When 提交表单，Then 显示错误提示并保存草稿 |
| **边界场景** | 边界条件测试 | Given 输入最大字符数，When 提交，Then 验证通过 |
| **兼容场景** | 不同环境兼容性 | Given 使用Chrome浏览器，When 打开页面，Then 功能正常 |

### 非功能验收标准
| 类型 | 指标 | 验收标准 |
|------|------|----------|
| **性能** | 响应时间 | API响应时间≤500ms（95%请求） |
| **安全性** | 安全要求 | 密码传输必须加密，符合TLS 1.2标准 |
| **可用性** | 系统可用性 | 系统月度可用性≥99.5% |
| **兼容性** | 浏览器支持 | 支持Chrome 90+、Firefox 88+、Safari 14+ |

## 9.2 验收标准模板

### Gherkin语法模板
```gherkin
功能: [功能名称]
  作为 [角色]
  我想要 [目标]
  以便于 [价值]

  场景: [场景名称]
    假如 [前提条件]
    当 [操作]
    那么 [预期结果]
```

### 结构化模板
| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| **场景描述** | 是 | 测试场景简要描述 | 用户登录成功 |
| **前置条件** | 是 | 执行前的系统状态 | 用户账号已注册且未锁定 |
| **测试数据** | 否 | 测试使用的数据 | 用户名: test@example.com, 密码: Test123! |
| **操作步骤** | 是 | 具体操作序列 | 1. 打开登录页面 2. 输入用户名密码 3. 点击登录按钮 |
| **预期结果** | 是 | 操作后的预期状态 | 1. 跳转到首页 2. 显示欢迎消息 3. 记录登录日志 |
| **验收依据** | 否 | 验收的参考标准 | 用户需求文档第3.2节 |

## 9.3 测试用例关联

### 关联类型
| 关联级别 | 关联对象 | 同步机制 |
|----------|----------|----------|
| **一对一** | 单个测试用例 | 自动创建测试用例草稿 |
| **一对多** | 多个测试用例 | 根据场景自动拆分 |
| **多对一** | 组合测试用例 | 合并多个验收标准 |

### 同步规则
| 变更类型 | 同步动作 | 通知机制 |
|----------|----------|----------|
| **验收标准新增** | 自动创建测试用例草稿 | 通知测试负责人 |
| **验收标准修改** | 更新关联测试用例 | 标记测试用例需更新 |
| **验收标准删除** | 标记关联测试用例作废 | 通知测试负责人确认 |
| **测试结果更新** | 更新验收标准状态 | 更新需求完成状态 |

## 9.4 验收状态管理

### 状态流转
| 状态 | 说明 | 允许操作 |
|------|------|----------|
| **草稿** | 初始创建状态 | 编辑、删除、关联测试用例 |
| **评审中** | 正在评审 | 评论、建议修改、批准 |
| **已批准** | 评审通过 | 关联测试用例、执行测试 |
| **测试中** | 测试执行中 | 更新测试结果、记录问题 |
| **已通过** | 所有测试通过 | 标记需求完成、生成报告 |
| **未通过** | 测试未通过 | 记录缺陷、重新测试 |
| **已废弃** | 不再使用 | 查看历史、恢复 |

### 状态检查规则
| 检查项 | 检查条件 | 处理动作 |
|--------|----------|----------|
| **完整性检查** | 必填字段是否完整 | 阻止状态流转 |
| **一致性检查** | 与需求描述是否一致 | 发出警告提示 |
| **可测试性检查** | 是否具备可测试条件 | 建议修改 |
| **依赖性检查** | 依赖条件是否满足 | 阻止状态流转 |

## 9.5 验收执行与跟踪

### 执行计划
| 执行阶段 | 执行时机 | 负责人 |
|----------|----------|--------|
| **单元测试** | 开发完成后 | 开发工程师 |
| **集成测试** | 模块集成后 | 测试工程师 |
| **系统测试** | 版本发布前 | 测试团队 |
| **用户验收** | 正式发布前 | 产品经理/客户 |

### 跟踪指标
| 指标 | 计算公式 | 目标值 |
|------|----------|--------|
| **验收通过率** | 已通过验收标准数 / 总验收标准数 | ≥95% |
| **首次通过率** | 首次测试通过数 / 总测试执行数 | ≥80% |
| **缺陷密度** | 发现的缺陷数 / 验收标准数 | ≤0.1 |
| **验收周期** | 从创建到通过的平均时间 | ≤3天 |

## 9.6 AI辅助验收标准

### AI生成能力
| 功能 | 输入 | 输出 |
|------|------|------|
| **自动生成** | 需求描述、用户故事 | Gherkin格式验收标准 |
| **场景识别** | 业务流程描述 | 正向、异常、边界场景 |
| **数据建议** | 业务规则、数据类型 | 测试数据建议 |
| **完整性检查** | 现有验收标准 | 缺失场景建议 |

### AI验证能力
| 功能 | 验证内容 | 输出结果 |
|------|----------|----------|
| **可测试性验证** | 验收标准描述 | 可测试性评分 |
| **一致性验证** | 验收标准 vs 需求 | 一致性报告 |
| **覆盖度分析** | 验收标准覆盖范围 | 场景覆盖度百分比 |
| **冲突检测** | 多个验收标准之间 | 冲突点列表 |

## 9.7 报表与分析

### 验收状态报表
| 报表类型 | 显示内容 | 分析维度 |
|----------|----------|----------|
| **状态分布** | 各状态验收标准数量 | 模块、迭代、负责人 |
| **通过率趋势** | 验收通过率变化趋势 | 时间、项目、团队 |
| **缺陷分析** | 验收未通过原因分析 | 缺陷类型、严重程度 |
| **效率分析** | 验收周期分析 | 负责人、需求复杂度 |

### 质量指标
| 指标 | 监控频率 | 预警阈值 |
|------|----------|----------|
| **验收积压** | 每日 | 未开始验收>10个 |
| **超期验收** | 每日 | 验收周期>5天 |
| **高缺陷率** | 每周 | 缺陷密度>0.2 |
| **低通过率** | 每周 | 首次通过率<70% |

## 9.8 集成与自动化

### 测试工具集成
| 集成工具 | 集成方式 | 同步内容 |
|----------|----------|----------|
| **JUnit/TestNG** | API接口调用 | 测试用例、测试结果 |

| **Postman/Insomnia** | 集合导出 | API测试集合 |

### 自动化流水线
| 流水线阶段 | 触发条件 | 执行动作 |
|------------|----------|----------|
| **代码提交** | Git Push | 运行关联单元测试 |
| **构建完成** | CI构建成功 | 运行集成测试 |
| **部署完成** | 部署到测试环境 | 运行系统测试 |
| **发布前** | 生产发布前 | 运行验收测试 |
