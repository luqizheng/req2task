# 核心功能流程

## 3.1 需求创建流程

```
1. 项目经理创建项目
2. 添加项目成员并分配角色
3. 创建原始需求收集（记录需求会议/调研）
4. 录入原始需求（客户原话/描述）
5. AI 分析原始需求并生成结构化需求条目（用户故事+验收条件）
6. 需求评审（审查/批准/拒绝）
7. AI 自动分解需求为任务并估算工时
8. 任务自动分配给 AI 机器人执行
9. 实时跟踪任务进度与完成情况
```

## 3.2 AI 辅助能力

- **需求生成**：用户输入模糊想法 → AI 生成结构化需求文档
- **评审预判**：AI 自动评分，预判评审争议点
- **工作估算**：基于历史数据精确估算人天
- **任务分解**：自动拆解功能点 → 生成任务列表
- **变更评估**：AI 评估变更影响范围，更新工作量

## 3.3 变更追溯

- 任何需求/任务的变更记录到变更日志
- 支持查看历史变更
- 支持变更对比
