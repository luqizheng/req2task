# 核心功能流程

## 3.1 系统架构概述

### 3.1.1 模块架构图

```
┌─────────────────────────────────────────────────────────────────┐
│                        前端展示层 (Web)                          │
├─────────────────────────────────────────────────────────────────┤
│  项目看板  │  需求管理  │  任务跟踪  │  AI助手  │  报表中心     │
└──────┬─────────────────────────────────────────────────────────┘
       │ HTTP/WebSocket
┌──────▼─────────────────────────────────────────────────────────┐
│                        API网关层 (Gateway)                       │
└──────┬─────────────────────────────────────────────────────────┘
       │
┌──────▼─────────────────────────────────────────────────────────┐
│                     业务逻辑层 (Service)                        │
├──────────┬──────────┬──────────┬──────────┬──────────────────┤
│ 项目管理  │ 需求管理  │ 任务管理  │ AI服务   │ 变更追溯         │
│ 服务      │ 服务      │ 服务      │ 集成     │ 服务             │
└──────┬────────────┬─┴─────┬────┴─┴────────┴──────────────────┘
       │            │       │
┌──────▼────┐ ┌─────▼─────┐ ┌▼────────────┐
│  PostgreSQL│ │  Redis    │ │ ChromaDB    │

└────────────┘ └───────────┘ └─────────────┘
```

### 3.1.2 核心模块职责

| 模块 | 职责边界 | 核心能力 |
|------|----------|----------|
| 项目管理 | 项目生命周期控制 | 创建/配置/归档项目、管理成员、权限分配 |
| 需求管理 | 需求全生命周期 | 原始需求收集、结构化转换、评审审批、版本控制 |
| 任务管理 | 任务执行跟踪 | 任务分解、分配、状态流转、依赖管理 |
| AI服务 | 智能辅助决策 | 需求生成、任务拆解、工作估算、相似推荐 |
| 变更追溯 | 变更全程记录 | 变更日志、版本对比、影响分析、基线管理 |
| 验收管理 | 质量把控 | 验收条件定义、测试关联、执行跟踪 |
| 报表中心 | 数据可视化 | 进度看板、燃尽图、趋势分析 |

### 3.1.3 模块依赖关系

```
项目管理 ─┬─> 需求管理
          ├─> 任务管理
          └─> 报表中心

需求管理 ─┬─> AI服务（需求生成）
          ├─> 验收管理（关联验收条件）
          └─> 变更追溯（版本记录）

任务管理 ─┬─> AI服务（任务拆解）
          ├─> 依赖管理
          └─> 变更追溯（任务变更）

验收管理 ─> 变更追溯（验收结果记录）
```

## 3.2 核心业务流程

### 3.2.1 需求创建与转换流程

#### 主流程：原始需求 → 结构化需求

```
┌─────────────┐
│ 启动项目    │─────────────────────────┐
└──────┬──────┘                         │
       │ 创建原始需求收集                 │
       ▼                                │
┌─────────────┐                         │
│ 需求收集    │                         │
└──────┬──────┘                         │
       │ 录入原始需求                     │
       ▼                                │
┌─────────────┐     ┌───────────────────▼───────────────┐
│ AI分析      │────▶│ 生成结构化需求                      │
│ 原始需求    │     │ 1. 用户故事（角色+目标+价值）         │
└─────────────┘     │ 2. 验收条件（Given-When-Then）      │
                    └───────────────────────────────────┘
```

#### 流程节点详细说明

| 节点 | 输入 | 处理逻辑 | 输出 | 异常处理 |
|------|------|----------|------|----------|
| 启动项目 | 项目基本信息 | 验证名称唯一性、初始化项目结构 | 项目ID | 名称冲突：提示重命名 |
| 需求收集 | 收集类型、会议纪要 | 创建收集记录、关联收集人 | 收集ID | 必填字段缺失：阻止提交 |
| 原始需求录入 | 需求内容、来源 | 持久化存储、状态标记pending | 原始需求ID | 内容为空：提示必填 |
| AI需求生成 | 原始需求文本 | 调用LLM生成结构化内容 | 用户故事+验收条件 | API超时：返回草稿、允许手动编辑 |
| 评审审批 | 生成的需求 | 状态流转：draft→reviewed→approved/rejected | 审批结果 | 审批超时：自动升级通知 |

#### 分支流程：AI生成失败

```
AI生成请求
     │
     ▼
┌─────────┐     超时/错误      ┌────────────────┐
│ 发送请求 │──────────────────▶│ 返回错误提示    │
└────┬────┘                    │ 1. 允许手动创建  │
     │                         │ 2. 支持重试      │
     │ 成功                     └────────────────┘
     ▼
┌─────────┐
│ 接收结果 │
└────┬────┘
     │ 生成失败
     ▼
┌────────────────┐
│ 标记状态为失败  │
│ 保留原始输入    │
│ 通知用户        │
└────────────────┘
```

### 3.2.2 任务分解与分配流程

#### 主流程：需求 → 任务 → 分配

```
┌─────────────┐
│ 需求评审通过 │───────────────────────┐
└──────┬──────┘                       │
       │ 触发任务分解                   │
       ▼                              │
┌─────────────┐     ┌─────────────────▼───────────────┐
│ AI任务分解  │────▶│ 生成任务列表                     │
└─────────────┘     │ 1. 任务标题                      │
                    │ 2. 任务描述                      │
                    │ 3. 预估工时                      │
                    │ 4. 优先级                        │
                    └─────────────┬───────────────────┘
                                  │ 存在依赖关系
                                  ▼
                    ┌─────────────────────────────┐
                    │ 建立任务依赖                 │
                    │ 1. 前置任务标记               │
                    │ 2. 依赖关系可视化             │
                    └─────────────┬─────────────────┘
                                  │ 分配任务
                                  ▼
                    ┌─────────────────────────────┐
                    │ 自动/手动分配               │
                    │ 1. 人类开发者                │
                    │ 2. AI机器人                 │
                    └─────────────────────────────┘
```

#### 任务分配规则

| 分配方式 | 触发条件 | 分配逻辑 | 执行者 |
|----------|----------|----------|--------|
| 自动分配 | AI任务类型 | 根据任务类型分配给对应AI机器人 | 系统自动 |
| 手动分配 | 人类任务 | 项目经理/产品经理指定 | 人工操作 |
| 负载均衡 | 多个可执行人 | 选择当前负载最低的成员 | 系统辅助 |
| 技能匹配 | 特定技术要求 | 匹配具备相应技能的成员 | 系统辅助 |

#### 异常处理：任务分配冲突

```
检测到任务冲突
     │
     ▼
┌──────────────────┐
│ 冲突类型判断      │
└────┬─────────────┘
     │
     ├─ 人员冲突（同一成员被多次分配同时段任务）
     │       │
     │       ▼
     │ ┌──────────────────┐
     │ │ 方案1：延后冲突任务│
     │ │ 方案2：重新分配   │
     │ │ 方案3：增加资源   │
     │ └──────────────────┘
     │
     └─ 依赖冲突（前置任务未完成）
             │
             ▼
      ┌──────────────────┐
      │ 阻塞任务标记      │
      │ 等待前置完成      │
      │ 发送通知          │
      └──────────────────┘
```

### 3.2.3 需求变更流程

#### 主流程：变更发起 → 影响评估 → 审批执行

```
┌─────────────┐
│ 发现变更需求 │───────────────────────────┐
└──────┬──────┘                           │
       │ 发起变更申请                      │
       ▼                                  │
┌─────────────┐     ┌─────────────────────▼───────────────┐
│ 填写变更信息 │────▶│ AI影响评估                           │
└─────────────┘     │ 1. 影响范围分析                       │
                    │ 2. 工作量变化                         │
                    │ 3. 依赖关系影响                        │
                    │ 4. 风险评估                           │
                    └─────────────┬─────────────────────────┘
                                  │ 评估通过
                                  ▼
                    ┌─────────────────────────────┐
                    │ 变更审批                     │
                    │ 1. 常规变更：产品经理审批     │
                    │ 2. 重大变更：多级审批         │
                    └─────────────┬───────────────┘
                                  │ 审批通过
                                  ▼
                    ┌─────────────────────────────┐
                    │ 执行变更                     │
                    │ 1. 更新需求版本               │
                    │ 2. 调整相关任务               │
                    │ 3. 记录变更日志               │
                    │ 4. 通知相关方                 │
                    └─────────────────────────────┘
```

#### 变更影响评估维度

| 评估维度 | 分析内容 | 输出结果 |
|----------|----------|----------|
| 范围影响 | 涉及的需求、模块范围 | 影响清单 |
| 工作量影响 | 新增/减少工时 | 工时差异报告 |
| 进度影响 | 对里程碑的影响 | 进度影响分析 |
| 依赖影响 | 上下游任务影响 | 依赖影响图 |
| 风险影响 | 技术风险、业务风险 | 风险评估报告 |

#### 变更分级规则

| 变更级别 | 判定条件 | 审批要求 |
|----------|----------|----------|
| L1轻微 | 工时变化<10%、不涉及依赖 | 自动通过 |
| L2一般 | 工时变化10-30%、小范围影响 | 产品经理审批 |
| L3重大 | 工时变化>30%、跨模块影响 | 多级审批 |
| L4紧急 | 阻断性问题修复 | 简化审批、先执行后补流程 |

## 3.4 关键业务规则与约束

### 3.4.1 数据约束

| 约束类型 | 约束规则 | 违反处理 |
|----------|----------|----------|
| 唯一性约束 | 项目名称在同一租户下唯一 | 阻止创建、提示重命名 |
| 必填约束 | 需求标题、描述必填 | 阻止提交 |
| 格式约束 | 日期格式、工时数字格式 | 格式校验 |
| 长度约束 | 标题≤255字符、描述≤5000字符 | 前端限制、后端校验 |
| 范围约束 | 优先级枚举、工时>0 | 枚举校验 |

### 3.4.2 流程约束

| 约束规则 | 约束内容 | 违反处理 |
|----------|----------|----------|
| 状态前置约束 | 需求未通过评审不可分配任务 | 阻止分配操作 |
| 依赖约束 | 阻塞任务不可设置为已完成 | 状态流转校验 |
| 权限约束 | 无编辑权限不可修改数据 | 403禁止访问 |
| 数量约束 | 单个项目最大成员数100人 | 阻止新增 |
| 层级约束 | 需求嵌套最多3层 | 阻止创建第4层 |

### 3.4.3 AI服务约束

| 约束类型 | 约束规则 | 处理方式 |
|----------|----------|----------|
| 请求频率 | 单用户每分钟最多10次AI请求 | 频率限制 |
| 内容长度 | 单次请求内容≤4000字符 | 前端截断 |
| 超时处理 | AI请求超时时间30秒 | 返回错误、允许重试 |
| 降级策略 | AI服务不可用时提供手动模式 | 降级到手动输入 |
| 敏感词过滤 | 检测并过滤敏感内容 | 拒绝请求并提示 |

### 3.4.4 变更控制约束

| 约束规则 | 约束内容 | 控制方式 |
|----------|----------|----------|
| 基线锁定 | 基线创建后需求不可直接修改 | 需创建新版本 |
| 版本号递增 | 版本号必须递增 | 自动递增校验 |
| 变更记录 | 所有变更必须记录日志 | 审计追踪 |
| 审批时效 | 审批超时自动升级 | 定时任务触发 |

## 3.5 异常处理机制

### 3.5.1 异常分类

| 异常类型 | 异常场景 | 处理策略 |
|----------|----------|----------|
| 业务异常 | 状态不允许流转、数据校验失败 | 返回错误信息、提示修正 |
| 权限异常 | 无权访问、无权操作 | 403禁止、提示权限不足 |
| 依赖异常 | 阻塞任务未完成、循环依赖 | 阻止操作、提示依赖关系 |
| 系统异常 | 数据库错误、服务不可用 | 记录日志、降级处理 |
| AI异常 | AI服务超时、返回格式错误 | 降级到手动模式、记录异常 |

### 3.5.2 全局异常处理流程

```
捕获异常
    │
    ▼
┌──────────────────┐
│ 异常类型判断      │
└────┬─────────────┘
     │
     ├─ 业务异常
     │       │
     │       ▼
     │ ┌──────────────────┐
     │ │ 返回业务错误码    │
     │ │ 包含错误信息和建议│
     │ └──────────────────┘
     │
     ├─ 系统异常
     │       │
     │       ▼
     │ ┌──────────────────┐
     │ │ 记录详细日志     │
     │ │ 返回友好错误信息  │
     │ │ 触发告警通知     │
     │ └──────────────────┘
     │
     └─ AI异常
             │
             ▼
      ┌──────────────────┐
      │ 降级到手动模式   │
      │ 记录AI调用日志   │
      │ 通知相关人员     │
      └──────────────────┘
```

### 3.5.3 数据一致性保证

| 场景 | 保证机制 | 实现方式 |
|------|----------|----------|
| 多表更新 | 事务控制 | 数据库事务 |
| 状态流转 | 乐观锁/悲观锁 | 版本号控制 |
| 并发修改 | 冲突检测 | 最后写入胜出或人工确认 |
| 数据同步 | 最终一致性 | 消息队列异步同步 |
| 缓存一致性 | 缓存失效 | Redis缓存失效策略 |

## 3.6 模块间交互协议

### 3.6.1 同步调用关系

| 调用方 | 被调用方 | 调用方式 | 接口类型 |
|--------|----------|----------|----------|
| 前端 | API网关 | HTTP REST | 同步请求/响应 |
| API网关 | 业务服务 | RPC | 同步调用 |
| 业务服务 | AI服务 | HTTP REST | 同步调用+超时控制 |
| 业务服务 | 数据库 | ORM | 同步操作 |

### 3.6.2 异步事件关系

| 事件类型 | 触发者 | 订阅者 | 事件内容 |
|----------|--------|--------|----------|
| 需求状态变更 | 需求服务 | 任务服务、通知服务 | 需求ID、新状态、变更人 |
| 任务完成 | 任务服务 | 需求服务、报表服务 | 任务ID、完成时间 |
| AI生成完成 | AI服务 | 需求服务 | 生成结果、关联需求ID |
| 变更审批通过 | 审批服务 | 需求服务、通知服务 | 变更ID、审批结果 |

## 3.7 性能与安全约束

### 3.7.1 性能要求

| 指标 | 目标值 | 测量方式 |
|------|--------|----------|
| API响应时间 | P95<500ms | APM监控 |
| 页面加载时间 | <3秒 | 前端监控 |
| AI生成时间 | <30秒 | 服务端日志 |
| 并发用户数 | ≥100 | 负载测试 |

### 3.7.2 安全要求

| 安全类型 | 控制措施 |
|----------|----------|
| 认证授权 | JWT令牌、RBAC权限控制 |
| 数据隔离 | 租户隔离、项目级数据隔离 |
| 敏感数据 | 加密存储、脱敏展示 |
| SQL注入 | 参数化查询、ORM防护 |
| XSS攻击 | 输入过滤、输出编码 |
| CSRF攻击 | Token验证 |
