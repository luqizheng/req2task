# 产品需求文档 - 需求转任务系统（req2task）

## 文档结构

本产品需求文档已拆分为多个模块化文件，便于维护和阅读：

1. **[系统概述](system-overview.md)** - 产品定位、目标用户、核心价值、使用场景
2. **[核心模型](core-models.md)** - 层级结构、实体关系、字段定义
3. **[功能流程](functional-flows.md)** - 需求创建流程、AI辅助能力、变更追溯
4. **[非功能性需求](non-functional-requirements.md)** - 性能、安全、可用性、可维护性等需求
5. **[需求优先级管理](requirement-priority.md)** - MoSCoW方法、业务价值矩阵、依赖管理
6. **[需求版本控制](requirement-versioning.md)** - 版本管理、基线管理、回滚机制
7. **[验收标准管理](acceptance-criteria-management.md)** - 验收标准分类、测试关联、状态管理
8. **[权限与工作流](permission-workflow.md)** - 角色权限、工作流引擎、审批流程
9. **[API设计](api-design.md)** - 项目管理、需求管理、任务管理等API接口
10. **[数据模型](data-models.md)** - 实体用途与核心字段总结
