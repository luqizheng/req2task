# 需求版本控制

## 8.1 版本管理模型

### 版本号规范
| 版本类型 | 格式 | 示例 | 说明 |
|----------|------|------|------|
| **主版本** | vX.0.0 | v2.0.0 | 重大变更，可能不兼容 |
| **次版本** | vX.Y.0 | v2.1.0 | 功能增强，向后兼容 |
| **修订版本** | vX.Y.Z | v2.1.3 | Bug修复，微小改进 |

### 版本生命周期
| 阶段 | 状态 | 允许操作 |
|------|------|----------|
| **草稿** | Draft | 创建、编辑、删除 |
| **评审中** | Review | 评论、建议修改 |
| **已批准** | Approved | 基线化、关联任务 |
| **已发布** | Released | 只读、创建新版本 |
| **已废弃** | Deprecated | 查看历史、恢复 |

## 8.2 基线管理

### 基线创建规则
| 基线类型 | 触发条件 | 保留策略 |
|----------|----------|----------|
| **迭代基线** | 每个迭代开始前 | 保留最近10个迭代 |
| **发布基线** | 每次产品发布前 | 永久保留 |
| **里程碑基线** | 项目关键里程碑 | 永久保留 |
| **临时基线** | 重大变更前 | 保留30天 |

### 基线内容
| 组件 | 包含内容 | 版本信息 |
|------|----------|----------|
| **需求文档** | 需求描述、用户故事、验收条件 | 完整版本历史 |
| **关联任务** | 任务列表、分配情况、进度 | 快照状态 |
| **依赖关系** | 需求间依赖、任务依赖 | 关系图谱 |
| **审批记录** | 评审意见、批准记录 | 时间戳、审批人 |

## 8.3 版本对比与差异分析

### 对比维度
| 对比类型 | 对比内容 | 输出格式 |
|----------|----------|----------|
| **文本对比** | 需求描述、用户故事内容 | 行级差异高亮 |
| **字段对比** | 优先级、状态、负责人等字段 | 表格对比 |
| **结构对比** | 用户故事、验收条件结构变化 | 树形结构对比 |
| **依赖对比** | 依赖关系变化 | 网络图对比 |

### 差异分析报告
| 分析项 | 检测规则 | 严重等级 |
|--------|----------|----------|
| **内容变更** | 文本相似度<90% | 中 |
| **优先级变更** | 优先级等级变化 | 高 |
| **范围变更** | 新增/删除用户故事 | 高 |
| **依赖变更** | 依赖关系增减 | 中 |

## 8.4 版本回滚机制

### 回滚类型
| 回滚场景 | 回滚范围 | 影响评估 |
|----------|----------|----------|
| **单需求回滚** | 单个需求到指定版本 | 低影响，自动评估依赖 |
| **模块回滚** | 整个功能模块回滚 | 中影响，需要人工确认 |
| **项目回滚** | 项目整体回滚到基线 | 高影响，需要审批 |

### 回滚流程
1. **回滚申请**：提交回滚申请，说明原因
2. **影响分析**：系统自动分析影响范围
3. **审批流程**：项目经理审批回滚操作
4. **执行回滚**：系统执行回滚，创建回滚记录
5. **通知相关方**：通知所有受影响人员

## 8.5 分支与合并

### 分支策略
| 分支类型 | 用途 | 生命周期 |
|----------|------|----------|
| **主分支** | 稳定版本，用于发布 | 长期存在 |
| **特性分支** | 开发新功能需求 | 功能完成后合并 |
| **修复分支** | 紧急Bug修复 | 修复后合并 |
| **实验分支** | 技术探索、原型验证 | 探索完成后删除 |

### 合并机制
| 合并类型 | 触发条件 | 冲突处理 |
|----------|----------|----------|
| **快速合并** | 无冲突，线性历史 | 自动合并 |
| **三方合并** | 有冲突需要解决 | 人工解决冲突 |
| **压缩合并** | 合并多个提交为一个 | 保持历史整洁 |
| **变基合并** | 重新应用提交到新基础 | 线性历史 |

## 8.6 版本历史与审计

### 历史记录
| 记录类型 | 记录内容 | 保留期限 |
|----------|----------|----------|
| **版本创建** | 创建人、时间、版本号 | 永久 |
| **版本修改** | 修改内容、修改人、时间 | 永久 |
| **版本发布** | 发布时间、发布人、发布说明 | 永久 |
| **版本回滚** | 回滚原因、回滚人、时间 | 永久 |

### 审计追踪
| 审计项 | 审计内容 | 审计频率 |
|--------|----------|----------|
| **版本完整性** | 版本链是否完整 | 每日 |
| **权限合规** | 版本操作是否符合权限 | 实时 |
| **变更合规** | 变更是否符合流程 | 实时 |
| **数据一致性** | 版本数据是否一致 | 每周 |

## 8.7 存储与备份

### 存储策略
| 存储类型 | 存储内容 | 存储期限 |
|----------|----------|----------|
| **热存储** | 最近30天版本 | 高性能存储 |
| **温存储** | 30天-1年版本 | 标准存储 |
| **冷存储** | 1年以上版本 | 低成本存储 |
| **归档存储** | 已发布版本基线 | 永久存储 |

### 备份策略
| 备份类型 | 备份频率 | 恢复时间目标 |
|----------|----------|----------------|
| **全量备份** | 每周一次 | ≤4小时 |
| **增量备份** | 每日一次 | ≤2小时 |
| **实时备份** | 持续同步 | ≤30分钟 |
| **异地备份** | 每日一次 | ≤24小时 |

## 8.8 API接口设计

### 版本管理API
| 接口 | 方法 | 功能描述 |
|------|------|----------|
| `/requirements/:id/versions` | GET | 获取需求版本列表 |
| `/requirements/:id/versions/:version` | GET | 获取特定版本内容 |
| `/requirements/:id/versions` | POST | 创建新版本 |
| `/requirements/:id/versions/:version/compare` | GET | 版本对比 |
| `/requirements/:id/versions/:version/restore` | POST | 恢复到指定版本 |

### 基线管理API
| 接口 | 方法 | 功能描述 |
|------|------|----------|
| `/projects/:id/baselines` | GET | 获取项目基线列表 |
| `/projects/:id/baselines` | POST | 创建新基线 |
| `/projects/:id/baselines/:baselineId` | GET | 获取基线详情 |
| `/projects/:id/baselines/:baselineId/restore` | POST | 恢复到基线 |
