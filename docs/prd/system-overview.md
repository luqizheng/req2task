# 系统概述

## 1.1 产品定位

需求转任务（req2task）是一个软件需求管理系统，支持需求全生命周期管理、多维度信息关联、AI 辅助需求生成、变更追溯、项目进度可视化和项目知识库构建，为项目经理提供决策支持。

## 1.2 目标用户与痛点

| 角色 | 核心痛点 | 解决方案价值 |
|------|----------|--------------|
| 产品经理 | 需求写不清、评审被怼、需求总变 | AI帮你写清晰、预判争议、自动追踪变更影响 |
| 开发工程师 | 需求模糊、验收扯皮、文档滞后 | 需求直接关联代码结构、自动生成单元测试、获得精确工作量 |
| 测试工程师 | 需求不可测、验收标准缺失 | 自动生成可测试的验收条件 |
| 技术负责人 | 需求追溯难、技术债累积 | 变更自动评估技术影响、关联代码模块 |
| 项目经理 | 进度估算不准、任务分配困难 | AI自动拆解功能点、核算人天、生成排期建议 |

## 1.3 核心价值

**"让每一个需求，都能直接驱动开发，并精确估算人天"**

任务自动分配给 AI 机器人执行。

## 1.4 使用场景

- **需求撰写时**：PM输入模糊想法 → AI引导澄清 → 生成标准文档
- **需求评审前**：AI自动评分，预判问题点
- **开发开始时**：需求自动拆解为功能点 → 估算工作量 → 分配任务
- **需求变更时**：AI评估影响范围 → 更新工作量 → 通知相关方
- **新项目启动**：AI推荐历史相似需求的解决方案和工作量数据
